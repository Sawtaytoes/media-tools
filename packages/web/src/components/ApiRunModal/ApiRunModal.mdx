# ApiRunModal

The umbrella modal that opens when the user clicks **▶ Run via API** in `PageHeader`. Streams two SSE channels in parallel: the **parent** sequence (`/jobs/<id>/logs`) and the **child** step that's currently executing (`/jobs/<childId>/logs`). The latter is auto-discovered from `step-started` events on the parent stream.

Logs from the parent stream populate the modal's body. Per-step progress events from the child stream drive the inline progress bar. Cancelling sends `DELETE /jobs/<id>` for the parent — closing the modal mid-run also cancels.

## Behaviour

- **Open** → `apiRunModalAtom` set with `{ jobId, status }`. Status starts at `pending`/`running` and transitions through `completed` / `failed` / `cancelled` based on the SSE `done` event.
- **Parent SSE messages**:
  - `step-started` → captures `childJobId` + `stepId`, resets child progress, calls `setStepRunStatusAtom({ stepId, status: "running", jobId: childJobId })` to sync step card UI.
  - `step-finished` → clears child state, calls `setStepRunStatusAtom({ stepId, status: null })` to clear step card UI.
  - `{ line }` → appended to the log buffer (auto-scrolls to bottom via `scrollIntoView`).
  - `{ done, status }` → terminal state. Sets `runningAtom` false.
- **Child SSE messages**:
  - `progress` → merged into `childProgress`. The `percent` field drives the progress bar width.
- **Disconnect detection** → `useTolerantEventSource` calls `onPossiblyDisconnected` after a long silence; the parent handler treats this as a failure.
- **Close** → if status is still `running`, fires a best-effort `DELETE /jobs/<id>` before clearing state. Backdrop click does the same.
- **Cancel** → same `DELETE` request without closing the modal (lets the user watch the rest of the cancellation logs).

## Atom wiring

| Atom | Role |
|------|------|
| `apiRunModalAtom` | `null` ⇒ closed. `{ jobId, status }` ⇒ open and streaming |
| `runningAtom` | Set to `false` when a terminal SSE event arrives or close-during-run cancels |
| `setStepRunStatusAtom` | Updated with `{ stepId, status, jobId }` on `step-started`; cleared on `step-finished` |
| `promptModalAtom` | (Currently held as a setter; reserved for future prompt routing through this modal) |

## Server endpoints

| Endpoint | Purpose |
|----------|---------|
| `GET /jobs/<id>/logs` (SSE) | Parent + child log + progress streams |
| `DELETE /jobs/<id>` | Cancel a running job |
