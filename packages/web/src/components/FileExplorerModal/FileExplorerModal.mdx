# FileExplorerModal

A full-screen file browser modal with two modes: **browse** (default ‚Äî view and delete files) and **picker** (when `pickerOnSelect` is set ‚Äî confirm-folder mode used by `PathVariableCard`'s Browse flow). Includes an embedded `VideoModal` sub-component for in-browser playback of `.mkv`/`.mp4`/`.webm`/etc.

Sortable by name, duration, size, or modification time. Directories always sort first regardless of column. The delete-mode badge shows whether deletes go to the OS Recycle Bin or are permanent ‚Äî derived from `/files/delete-mode` per current path (some paths can't trash, e.g., network shares).

## Behaviour

- **Open** ‚Üí `fileExplorerAtom` set with `{ path, pickerOnSelect }`. `path` becomes the initial `currentPath`; `pickerOnSelect` decides browse vs picker mode.
- **Navigate** ‚Üí click a directory row or any breadcrumb segment. Both reset selection and re-fetch listing + delete-mode.
- **Sort** ‚Üí click column header. Same-column click flips direction; different column resets to ascending.
- **Select files** ‚Üí row checkbox (directories disabled). Header checkbox toggles all *files* (not directories ‚Äî they can't be deleted from this UI).
- **Delete** ‚Üí `Delete selected` posts `DELETE /files` with the array of paths. Failure list is shown in a window.alert.
- **Picker confirm** ‚Üí `üìå Use this folder` calls `pickerOnSelect(currentPath)`, closes the modal, and forwards the result to whoever opened it (typically `PathVariableCard`).
- **Video playback** ‚Üí clicking a video file row opens the inner `VideoModal`. It probes `/files/audio-codec`; if the audio codec is unsupported in browsers (`ac-3`, `dts`, `truehd`, ‚Ä¶) it streams via `/transcode/audio?codec=opus` instead of `/files/stream`.
- **Esc** ‚Üí closes the video sub-modal first if open, then the explorer.
- **window.openVideoModal** ‚Üí exposed globally so legacy result cards (e.g., from `LookupModal`) can pop the video player without owning the modal.

## Atom wiring

| Atom | Role |
|------|------|
| `fileExplorerAtom` | Visibility + initial path + picker callback. `null` ‚áí closed |

## Server endpoints

| Endpoint | Purpose |
|----------|---------|
| `GET /files/list?path=&includeDuration=1` | Directory listing (with video duration probing) |
| `GET /files/delete-mode?path=` | Reports `"trash"` vs `"permanent"` for the current path |
| `DELETE /files` body `{ paths }` | Bulk delete |
| `GET /files/stream?path=` | Direct video stream |
| `GET /files/audio-codec?path=` | Probes audio codec to decide stream vs transcode |
| `GET /transcode/audio?path=&codec=opus` | Transcoded stream when audio is browser-unsupported |
| `POST /files/open-external` body `{ path }` | OS-level "open in default player" (hidden when the server is containerized) |
| `GET /version` | Reports `isContainerized` so the external-open button can be hidden |
